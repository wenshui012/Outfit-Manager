# 穿搭管理器 Outfit Manager

SillyTavern 穿搭管理插件 —— 让 AI 不再为角色乱编服装。

当前版本：2.0.0

## 2.0 更新前请先备份

2.0 引入了分包存储、单品管理和套装方案。升级前请在旧版本中先导出一次完整备份，并保留原始备份文件；如果启用了后端存储，也建议同时备份后端 `data/` 目录。

升级后首次进入会自动迁移旧数据。迁移完成前不要反复刷新或同时打开多个 SillyTavern 页面写入同一份衣柜数据。

## 这个插件解决什么问题？

在 SillyTavern 角色扮演中，AI 经常自行编造角色的服装描写，导致人设崩坏。比如你设定的是穿西装的总裁，AI 可能下一句就给他换上了休闲T恤。

穿搭管理器通过将你预设的穿搭信息注入到 AI 的对话上下文中，确保 AI 严格按照指定的服装来描写角色。

## 功能特性

- **User / 角色独立穿搭**：User 和每个角色各有独立的穿搭管理
- **衣柜模式**：为同一角色设置多套穿搭，AI 根据场景标签自动匹配，场景转换时切换穿搭
- **单品管理**：发型、妆容、美甲、饰品、道具等可作为单品单独保存、分类、移动、复制和生成描述
- **套装方案**：在已有穿搭主体上保存一组单品搭配，点选单品只是临时选择，点击保存套装后才写入数据
- **三种注入方式**：纯文字 / 纯图片 / 文字+图片
- **三种注入位置**：系统提示末尾 / 上下文末尾 / 用户消息末尾（推荐）
- **场景标签系统**：为穿搭标记适用场景（如"外出"、"家居/睡前"），衣柜模式下 AI 按标签选择穿搭
- **通用衣柜**：通用衣柜常驻，可与角色单人衣柜配合使用
- **风格衣柜**：角色衣柜名称自由命名，可按风格分类（总裁类、校园类、年下类等）
- **图片转描述**：支持调用 API 将穿搭图片自动生成文字描述
- **分类管理**：穿搭支持自定义分类，方便整理
- **角色数据导入/导出**：支持按角色导出穿搭数据，方便分享和备份
- **可自定义模板**：注入给 AI 的提示词模板完全可编辑

## 安装方法

### 方法一：通过 Git 安装（推荐）

1. 打开 SillyTavern，进入 **扩展** → **安装扩展**
2. 输入本仓库的 Git 地址，点击安装

### 方法二：手动安装

1. 下载本仓库的 ZIP 文件
2. 解压后将文件夹放入 SillyTavern 的 `public/scripts/extensions/third-party/` 目录
3. 重启 SillyTavern

## 使用指南

### 基础用法（单套穿搭）

1. 点击侧栏的穿搭管理器按钮打开面板
2. 点击 **添加穿搭**，输入名称和文字描述（或上传图片）
3. 勾选激活这套穿搭
4. 开始聊天，AI 就会按照你设定的穿搭来描写

### 角色穿搭

1. 在角色选择器中选择或输入角色名
2. 为该角色添加穿搭
3. User 和角色的穿搭会合并后一起注入给 AI

### 衣柜模式（多套穿搭）

1. 为同一角色添加多套穿搭
2. 全部勾选激活
3. 建议为每套穿搭添加 **场景标签**（如"办公室"、"居家"、"外出"）
4. AI 会根据当前剧情场景选择最合适的穿搭，场景转换时自动切换

### 单品和套装

1. 切换到单品栏，添加发型、妆容、美甲、饰品、道具等单品
2. 单品可以独立分类，也可以批量导入并生成描述
3. 先选择一个穿搭主体，再点选要搭配的单品
4. 点击保存套装后，当前主体会记录这组单品搭配；未保存前只是临时选择

### 推荐设置

- **注入方式**：纯文字（最稳定）
- **注入位置**：用户消息末尾（Gemini/DeepSeek 兼容性最佳）
- **单套穿搭**：所有模型均稳定支持
- **衣柜模式**：文字模式下可用，纯图片模式下不建议使用衣柜

## 后端存储建议

默认 IndexedDB 本地存储已经可以使用。图片和衣柜数据较多、需要跨设备迁移，或希望减少浏览器本地存储压力时，建议安装配套后端 `Outfit-Manager-Server`。

使用后端前请确认 SillyTavern 已开启 server plugin，并定期备份后端插件目录下的 `data/`。如果后端不可用，插件会尽量回退到本地存储；恢复后端前建议先导出完整备份。

- 复现步骤：当前视角是 User、通用衣柜还是角色衣柜，是否处于预设模式，是否涉及单品/套装
- 导入导出、迁移、图片丢失类问题请说明是否已有备份，必要时提供脱敏后的数据结构片段
